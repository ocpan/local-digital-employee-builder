# 方法论动作清单（从真实项目实录蒸馏）

> 只保留**可复用的动作标签**——搭数字员工时，按这套顺序推进。

## 阶段 A · 资料收集（全景先行）

- 全景探索：先读整个 workspace/仓库目录结构，建立全局认知，再动手。
- 现有系统分析：分析要对接系统的代码结构与接口。
- 文档收集与转换：把需求/设计文档统一抓取、转 Markdown 归档。
- 数据对齐：把不同来源（文档、链接、代码）做精确映射，消除歧义。
- 知识库入库 + 完整性校验：内容入库后校验完整，不遗漏。
- 批量入库：文档多时用工具批量抓取正文，别手抄。

## 阶段 B · 需求文档与设计

- 需求文档编写：按模块拆分成产品需求说明书。
- 冲突检查：交叉比对不同来源文档的信息冲突。
- 接口设计：产出接口契约文档。
- 优先级决策：做需求优先级排序，写 P0 交付计划。

## 阶段 C · 实现与搭建

- 骨架搭建：先跑通最小核心，再逐步加能力。
- 配置驱动：差异收敛到配置文件，代码逻辑保持通用。
- 逐步验证：每完成一个模块就真实测试一轮。

## 阶段 D · 本地验证与持续迭代

- 本地验证：尽量在本地跑通，缩短反馈环。
- 全盘盘点：主动找遗漏模块补齐，而不是等人提。
- 问题闭环：发现的问题当场修，不积压。

## 贯穿全程的原则

- **文档先行**：先把需求和设计文档理清楚。
- **复用优先**：先看已有系统能给什么。
- **迭代验证**：每一步做完就验证。
- **全盘盘点**：定期全局检查。
- **一步步来**：按步骤执行，不跳步。
