# 申请一个飞书机器人（自建应用）· 图文步骤

目标：拿到 **App ID**（`cli_` 开头）和 **App Secret**，并让机器人能收发消息。
全程在飞书开发者后台点击完成，不用写代码。**做不下去就把这一步交给 Claude Code 陪你弄。**

---

## 第 1 步：创建企业自建应用

1. 浏览器打开飞书开发者后台：**https://open.feishu.cn/app**
2. 用你的飞书账号登录（必须是你**所在企业/团队**的账号）。
3. 点 **「创建企业自建应用」**，填名称（如"我的助手"）、图标、描述 → 创建。
4. 进入应用后，在 **「凭证与基础信息」** 页可以看到：
   - **App ID**（`cli_xxxxx`）
   - **App Secret**（点击显示，复制保存好，别外泄）

> 这两个就是第四步 Claude Code 要用的。

---

## 第 2 步：开启机器人能力

1. 左侧菜单 **「添加应用能力」** → 找到 **「机器人」** → 启用。
2. 启用后，机器人才能在飞书里被搜索到、被拉进群、收发消息。

---

## 第 3 步：申请消息权限（Scope）

左侧 **「权限管理」** → 搜索并开通下面这些（勾选后要"批量开通"）：

| 权限 | 作用 |
|---|---|
| `im:message` | 收发消息 |
| `im:message.p2p_msg:readonly` 或 `im:message:readonly` | 读取用户发来的消息内容 |
| `im:message:send_as_bot` | 以机器人身份发消息 |
| `contact:contact.base:readonly`（可选） | 显示提问人真实姓名（否则显示"同事"） |

> 加完权限要 **创建版本 → 发布 → 等管理员审核通过**，权限才生效（可能有几分钟延迟）。

---

## 第 4 步：配置事件订阅（关键！机器人靠这个"听见"消息）

1. 左侧 **「事件与回调」** → **「事件配置」**。
2. 订阅方式选 **「使用长连接接收事件」**（重要：不是 Webhook，长连接才配合 lark-cli）。
3. 添加事件：搜索并订阅 **`im.message.receive_v1`**（接收消息）。

> 这一步如果找不到，交给 Claude Code：把开发者后台页面截图发给它，它告诉你点哪里。

---

## 第 5 步：发布 + 配置可用范围

1. 左侧 **「版本管理与发布」** → **「创建版本」** → 填版本号和说明 → 提交发布。
2. 需要**企业管理员审核**（如果你自己是管理员，直接通过）。
3. 在 **「可用范围」** 里，把你自己（和要用机器人的同事）加进去，否则搜不到机器人。

---

## 完成后你应该有：

- ✅ App ID（`cli_xxxxx`）
- ✅ App Secret
- ✅ 机器人能力已开、消息权限已批、`im.message.receive_v1` 长连接事件已订阅、版本已发布

拿着这些回到 **`00-先看这里-START-HERE.md` 的第四步**，让 Claude Code 帮你接起来。

---

## 常见卡点

- **搜不到机器人** → 检查"可用范围"有没有加你自己 + 版本有没有发布通过。
- **发消息机器人不理** → 90% 是"事件订阅没配长连接"或"没发布/权限没生效"。
- **只能发不能读消息内容** → 缺 `im:message:readonly` 类权限，补上重新发布。
- **完全没头绪** → 把每一步的页面截图发给 Claude Code，它会精确告诉你点哪里。
